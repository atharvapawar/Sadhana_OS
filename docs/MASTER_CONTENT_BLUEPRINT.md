# Sadhana OS Master Content Blueprint v1

## North Star

**The Yogic River from Misidentification to Living Awareness**

Sadhana OS is a sacred knowledge system, not a random yoga blog. Every page, term, card, and practice should support one movement:

**Misidentification → Right Seeing → Purification & Foundation → Body-Prana Stabilization → Interiorization & Meditation → Living Realization**

The site must feel like one intelligent river: many streams of Sanatana/Yogic knowledge meeting a single direction, then becoming lived awareness in ordinary life.

## Fixed Main Sections

Do not add new top-level knowledge sections unless the whole architecture is deliberately revised. All knowledge should live inside these five main homes:

1. **Doctrine** — the highest view and liberation logic.
2. **Knowledge Roadmap** — the 18-stage sequence.
3. **Inner Science** — the human mechanism and traditional maps.
4. **Path & Practice** — safe application and practice lanes.
5. **Glossary** — definitions, distinctions, and relationship mapping.

## Doctrine

Doctrine answers: **What is the real problem? What is liberation? What is the correct view?**

Use Doctrine for:

- Atman, Brahman, Purusha, Sakshi, Chaitanya
- Misidentification with body, mind, memory, desire, fear, ego, and the separate knower
- Bondage, avidya, asmita, samskara, vasana, karma, dharma
- Moksha, kaivalya, jivanmukti, sahaja, living awareness
- Karma Yoga, Bhakti, Jnana, Raja Yoga, and their relationship
- Why practice does not create the Self; practice clarifies the instrument
- Why realization must express as humility, responsibility, service, and non-clinging

Doctrine should be clear and powerful, but not bloated. It should orient the seeker before technique.

## Inner Science

Inner Science answers: **What is the human instrument? What maps explain the body-prana-mind system?**

Use Inner Science for:

- Sharira Traya: gross, subtle, causal body
- Pancha Kosha: annamaya, pranamaya, manomaya, vijnanamaya, anandamaya
- Antahkarana: manas, chitta, buddhi, ahamkara
- Sakshi as witness, clearly distinguished from mental functions
- Pancha Prana and sub-vayus
- Nadi, chakra, kundalini as traditional subtle-body frameworks
- Pancha Mahabhuta, guna, dosha
- Sense organs, action organs, tattvas
- How lifestyle, breath, attention, and conduct affect the instrument

Inner Science must separate:

- Traditional Yogic Science map
- Practical meaning for sadhana
- Modern safety/medical caution
- Not one-to-one modern anatomical claim

## Path & Practice

Path & Practice answers: **What should a seeker safely do now?**

Use Path & Practice for:

- Beginner-safe daily routine
- Steady practitioner track
- Teacher-guided track
- Do-not-DIY advanced practices
- Practice sequencing
- Readiness signs
- Overpractice warnings
- What to reduce when anxiety, pressure, heat, insomnia, dissociation, or obsession appears
- Integration into work, relationship, speech, food, sleep, and responsibility

Practice content must always be safety-gated:

### Green Gate — Beginner-Safe

Study, reflection, yama, niyama, gentle asana, natural breath awareness, simple japa, gratitude, normal meditation, karma yoga reflection.

### Amber Gate — Guided

Long sittings, intensified pranayama, mantra discipline, chakra contemplation, extended tapas, deeper pratyahara, prolonged silence.

### Red Gate — Teacher-Only

Forced kumbhaka, strong bandha stacks, kundalini activation, khechari attempts, intense kriyas, extreme fasting, sleep deprivation, chakra forcing, energy manipulation, siddhi chasing.

## Glossary

Glossary answers: **What does this term mean, what is it related to, and what is it not?**

Each glossary term now uses this implemented template:

1. Core meaning
2. Beginner meaning
3. Deeper meaning
4. Related terms
5. Not the same as
6. Used in stage(s)
7. Safety label

Example:

**Atman**

- Simple meaning: the Self, pure awareness.
- Related: Sakshi, Chaitanya, Brahman.
- Not the same as: personality, ego, memory, thoughts, blissful mood.
- Used in: Right View, Human Architecture, Samadhi, Living Realization.

The current implementation lives in `src/data/glossary-relationships.ts` and renders on the existing Glossary page with search, category filters, stage links, and safety labels.

## Overlap Resolver Rules

Many traditional terms point toward the same liberation direction but should not be collapsed carelessly.

- **Atman** = Self / pure awareness.
- **Sakshi** = witness aspect recognized in experience.
- **Purusha** = consciousness principle in Yoga/Sankhya language.
- **Chitta** = mind-field; includes memory/impression field.
- **Manas** = sensory mind / coordinator.
- **Buddhi** = discrimination/intelligence.
- **Ahamkara** = I-maker / ego-function.
- **Kosha** = sheath/layer of experience.
- **Sharira** = body-level framework: gross, subtle, causal.
- **Tattva** = principle/category of manifestation.
- **Prana** = life-force principle in Yogic Science.
- **Vayu** = pranic function/direction.
- **Nadi** = subtle channel map.
- **Chakra** = subtle center map.
- **Dharana** = held one-pointedness.
- **Dhyana** = continuous meditation flow.
- **Samadhi** = absorption beyond normal separate-knower structure.
- **Moksha / Kaivalya / Jivanmukti / Sahaja** = related liberation languages; explain distinctions without turning them into competing goals.

## Content Acceptance Rules

Before adding content, ask:

1. Which main section does this belong to?
2. Which stage does it support?
3. Is it Doctrine, Inner Science, Practice, or Glossary?
4. Is it beginner-safe, guided, or teacher-only?
5. Does it repeat something already explained elsewhere?
6. Does it help the river sequence or create confusion?
7. Does it make a traditional claim look like modern medical fact?
8. Does it encourage unsafe DIY advanced practice?

## Implementation Rule

Every future implementation should be incremental:

1. Patch one section or one stage group.
2. Keep existing working behavior.
3. Run `npm run doctor`, `npm ci`, and `npm run build`.
4. Check mobile layout at 320px+.
5. Do not add pages unless the architecture explicitly approves them.



## Phase 6 Inner Science Concept Map Addendum

The Inner Science section now functions as the human-instrument map for the whole site. It must remain explanatory and relational, not a place for unsafe procedural instruction.

The six integrated maps are:

1. Sharira Traya — gross, subtle, causal body and witness distinction.
2. Pancha Kosha — experiential sheaths from body to bliss, with Self beyond the sheaths.
3. Antahkarana — manas, chitta, ahamkara, buddhi, and Sakshi distinction.
4. Prana, Vayu & Nadi — life-current and subtle-channel language.
5. Tattva, Guna & Dosha — manifestation, qualities, and constitutional tendencies.
6. Chakra & Granthi Language — subtle centers and knots, kept safety-gated.

Rule: every map must support the river sequence from misidentification to living awareness.


## Phase 8 Path & Practice Safety System

The Path & Practice section now translates the 18-stage river into readiness lanes: beginner-safe foundation, steady practitioner, teacher-guided depth, and protected do-not-DIY zones. Advanced practices remain visible as traditional knowledge but are protected by safety gates and safer alternatives.


## Production QA Rule

Every future content or UX phase must preserve the automated integrity checks introduced in Phase 11. The 18 stages, 5 arcs, stage intelligence cards, glossary relationship fields, and deployable SPA routes are now protected by `npm run content:check` and `npm run route:smoke`.

Before shipping, run:

```bash
npm ci
npm run check
```
